rend.material.color = new Color(Random.value, Random.value, Random.value);の意味って？

左辺の意味

まず、rend (Renderer) の、material ってやつの、 color を変更してねってのが左辺。
new Color( ) は、色を変えるときのおまじない*1みたいなもので、とりあえずこれを書いておけばおっけー。

右辺の意味

これから Color っていう言い方で赤色の要素を Random.value, 緑色の要素を Random.value, 青色の要素を Random.valueだけ含んだ色を持つよってのが右辺。

なんでColor()のなかに3回も繰り返してRandom.valueを書いてるの？

結論、Color()っていう関数が 赤色の要素、緑色の要素、青色の要素 っていう3つの数字*3*4を欲しがってるから。

ちなみにColor() は、説明を見る(関数にマウスカーソルをあわせる)と、こんな感じのが載ってる

※画像1 (タップで拡大)

これを翻訳してみると、

Constructs a new color with given r,g,b components and sets a to 1
指定されたr、g、b成分から新しい色を構築し、aを1に設定します。

となり、3つの数字と a を定義*4していることがわかる。

まとめ

rend.material.color = new Color(Random.value, Random.value, Random.value);は、ランダムな色を設定するコード。
そして "ランダムな色" を赤色の要素をランダムに、緑色の要素をランダムに、青色の要素をランダムに決めてるから3回連続でRandom.valueって書いてるってこと。

*1 しっかり解説すると、Rendererのmaterialのcolorという値*2 は"Color"っていう言い方で情報を伝えてほしいとunity側が言ってる。だから、 new でColorっていう言い方をするよと伝えて、メモリにColorという箱を用意する。でColorに情報を書く。難しいから理解しなくて大丈夫。おまじないだと思ってもらって平気
*2 ここでいう値をプログラミング用語でプロパティという
*3 光の三原色、RGB (Red, Green, Blue)に対応している
*4 実は4つ目の数字(alpha値。a値ともいう。)をもたせることで 透明度 の情報をもたせることもできる。a = 1で不透明。 a = 0 で完全に透明。半透明なオブジェクトとか作りたいときに使う。
